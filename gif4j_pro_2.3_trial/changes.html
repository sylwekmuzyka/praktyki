Gif4J PRO :: Change Log
Change Log
 Changes from Gif4J Library version 2.2 to Gif4J PRO version 2.3
  • Added new ExactQuantizer class which lets to quantize image(-s) to exact number of colors.
  • Added support for requantize and exactRequantize transform operations for more optimization possibilities.
  • Fixed set of minor and major bugs
  • Updated quantization algorithms implementations.
  • Improved library speed and reliability.
 Changes from Gif4J Library version 2.1 to Gif4J PRO version 2.2
  • Fixed set of minor and major bugs
  • Added support for applyFrame watermark operation.
  • Updated quantization algorithms implementations.
  • Improved library speed and reliability.
 Changes from Gif4J Library version 2.0 to Gif4J PRO version 2.1
  • Fixed set of minor and major bugs
  • Added support for crop transform operation.
  • Updated quantization algorithms implementations.
  • Improved library speed and reliability.
 Changes from Gif4J Library version 1.0 to Gif4J PRO version 2.0
  • Gif4J Library has been renamed to Gif4J PRO
  • Added a GifDecoder to load & decode GIF images from files and streams.
  • Added a GifTransformer to transform GIF images including resize, scale, rotate and flip.
  • Added possibility to watermark GIF images in toto.
  • Added possibility to extract, add and remove GIF Image Comments.
  • Added possibility to access in the random order and iterate GIF image frames.
  • Added possibility to encode BufferedImage(-s) and GifImage(-s) to a file directly.
  • Added possibility to retrieve GifFrame's position, size, delay time and disposal method (getX(), getY(), getHeight(), getWidth(), getDisposalMethod(), getDelay()).
  • Added possibility to set and get an interlace GifFrame's flag.
  • Added possibility to retrieve frame's image data and color model.
  • A couple of minor bug fixes.
Note: Now by default a new GifImage doesn't contain a Looping Application Extension Block at all which means that the result GIF image is iterated only 1 time. To set, for example, infinite looping you should call gifImage.setLoopNumber(0).
